## 🎯 核心价值主张

### 传统开发流程 vs AI 需求交付控制台

| 维度 | 传统方式 | 使用本工具 |
|------|---------|-----------|
| **产物管理** | 散落在各个目录，难以查找 | 按需求号自动聚合，一键索引 |
| **进度跟踪** | 口头汇报或手动统计 | 可视化工作流，实时状态更新 |
| **Agent 执行** | 命令行操作，日志不可见 | 图形化界面，SSE 实时输出 |
| **协作效率** | 文档版本混乱，评审困难 | 统一标准，结构化评审报告 |
| **质量保障** | 测试覆盖率不透明 | 自动化测试 + 代码评审双重把关 |

### 4. 🔒 三重安全保障机制

**企业级安全设计，保护你的代码资产**

#### 🛡️ 第一重：路径限制
- Runner 只允许读写当前工作区内的文件
- 防止越权访问敏感目录

#### 🛡️ 第二重：并发控制
- 需求级锁文件机制
- 避免多人同时修改同一需求导致冲突

#### 🛡️ 第三重：Hash 校验
- Markdown 保存时比较文件 hash
- 发现外部修改时阻止覆盖，防止数据丢失

**实际效果：**
```
⚠️ 检测到文件已被外部修改！
   当前 Hash: abc123...
   文件 Hash: def456...
   
   建议操作：
   1. 点击「刷新产物」查看最新内容
   2. 手动合并差异后再保存
```

### 5. ⚡ 灵活的 Agent Provider 机制

**支持多种执行模式，适配不同团队需求**

#### 模式 A：Codex CLI 自动化（推荐）
```bash
# 配置一次，永久使用
CODEX_COMMAND='codex exec -C {workspaceRoot} -'

# 页面点击「执行」即可
# 系统自动：生成 Prompt → 调用 Codex → 展示日志
```

#### 模式 B：手动执行
```bash
# 页面生成标准命令
/coding-design d=docs/172014/prd/analysis.md r=172014

# 复制到任意 Agent 环境执行
# 执行完成后点击「刷新产物」
```

#### 模式 C：自定义 Agent
```json
{
  "id": "my-agent",
  "name": "My Custom Agent",
  "command": ["my-agent", "run", "--file", "{promptFile}"]
}
```

**优势：**
- 不锁定特定 Agent 厂商
- 支持私有化部署的 Agent
- 便于团队内部工具集成

## 💼 典型用户画像

### 👤 用户 A：Tech Lead 张伟
**痛点：** 需要跟踪团队 10+ 个需求的进展，每周汇报耗时耗力

**解决方案：**
- 使用需求列表页面查看所有需求状态
- 导出进度报告，自动生成周报数据
- 通过颜色标签快速识别阻塞的需求

**收益：** 周报准备时间从 2 小时缩短到 15 分钟

---

### 👤 用户 B：架构师 李娜
**痛点：** 代码评审标准不统一，问题描述模糊

**解决方案：**
- 使用 `coding-review` 技能自动化评审
- 问题分级（严重/警告/建议）清晰明确
- 支持外部文档约束核对

**收益：** 评审效率提升 60%，问题修复率提高 40%

---

### 👤 用户 C：测试工程师 王强
**痛点：** 遗留系统测试覆盖率低，手工编写测试成本高

- 使用 `coding-junit` 技能自动生成单元测试

- 根据报告调整测试策略

**收益：** 测试覆盖率从 30% 提升到 75%，节省 80% 编写时间
