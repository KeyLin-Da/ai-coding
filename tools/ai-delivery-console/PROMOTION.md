### 标题：告别混乱！用这个可视化工具管理你的 AI 编程工作流

**正文：**

你是否遇到过这些场景？

1. **产物管理混乱**：PRD 在一个目录，技术方案在另一个地方，测试报告又不知道在哪...
2. **Agent 执行黑盒**：点击执行后，只能等待结果，中间发生了什么完全不知道
3. **协作效率低下**：团队成员各自为战，代码评审标准不统一，问题描述模糊

如果你有以上痛点，那么 **AI 需求交付控制台** 就是为你打造的！

### 🎯 这是什么？

一个面向本地工作区的 OpenSpec + 自定义技能可视化交付台，用于按需求号聚合：
- 📄 PRD 分析
- 🏗️ 技术方案
- ✅ OpenSpec 实施验证
- 🧪 单元测试报告
- 👨‍💻 代码评审报告

### ✨ 核心亮点

#### 1. 可视化工作流
每个需求的状态一目了然：PRD ✓ → 设计 🔄 → 验证 ⏳ → 评审 ⏳

#### 2. SSE 实时日志
告别黑盒执行！随时查看 Agent 的执行步骤、耗时和输出。

#### 3. 三重安全保障
路径限制 + 并发控制 + Hash 校验，保护你的代码资产。

#### 4. 灵活的 Agent 集成
支持 Codex CLI 自动化执行，也支持手动模式，还可以注册自定义 Agent。

### 🚀 快速开始

```bash
cd tools/ai-delivery-console
npm install
npm run server:dev
npm run dev
```

访问 http://127.0.0.1:5178 即可看到主界面。
